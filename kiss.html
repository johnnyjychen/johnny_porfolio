UX Research - KISS

Project Information

Overview: In this project, our group assisted our clients in conducting a range of UX research for the KISS app, which enabled users to log their sexual activities and share their sexual health status with partners. Our work included user interviews, surveys, comparative evaluations, heuristic evaluations, and user testing in a course.

Project Type: Group Project

Duration: 2/2024 - 4/2024

Skills: UX Research, User Interviews, Surveys, Comparative Evaluations, Heuristic Evaluations, User Testing, Presentation

Heuristic Evaluations

<<<<<<< HEAD We conducted several heuristic analyses of the KISS application. The analysis of the KISS application revealed both strengths and weaknesses. Positively, the well-structured user flow for the three major use cases tested ensures ease of use. The “New Scanning” page mimics a real-world clinic, making data input intuitive. Additionally, feedback and system status updates keep users informed. However, the app limits editing or deleting past records, restricting user control. Icons and page titles on the “New Kiss” and “New Screening” pages are unclear, causing confusion. The app also lacks aesthetic and minimalist design, making information difficult to read and navigate. To address these issues, we recommended enabling users to edit and delete records, improve icon clarity and page titles with descriptive text, and redesign visual elements like the calendar and color scheme. These changes would enhance navigation and readability while providing a cleaner, more user-friendly interface. ======= We conducted several heuristic analyses of the KISS application. User Testing

<<<<<<< HEAD To enhance feedback on the KISS app's features, we conducted user testing on more users to observe behaviors and gathering feedback. User interviews revealed challenges in the activity addition process and icon ambiguity, especially for logging encounters and navigating QR codes. Recommendations include adding a prominent "Add New" button, clearer icons, and improved QR code navigation and accessibility. For screening results, users struggled with checkbox symbols for STI results and the lack of physician names in dropdown menus. Refinements should include intuitive checkboxes, detailed physician information, and a simplified facility addition process with prompt feedback. STI notifications posed difficulties with locating the send button and understanding the interface. Improving email visibility, clarifying messaging context, and redesigning the interface can address these issues. ======= To enhance feedback on the KISS app's features, we conducted user testing on more users to observe behaviors and gathering feedback.
